# 读书划线 → 康奈尔笔记

将阅读划线/摘录智能转化为结构化的康奈尔笔记（Cornell Notes），
帮助用户从「被动划线」升级到「主动学习」。

## 康奈尔笔记法简介

康奈尔笔记是美国康奈尔大学 Walter Pauk 教授提出的笔记系统，将一页纸分为四个区域：

```
┌──────────────────────────────────┐
│  标题：书名 · 作者 · 日期 · 章节    │
├────────────┬─────────────────────┤
│  线索栏     │  笔记栏              │
│  (Cue)     │  (Notes)            │
│            │                    │
│  · 关键词   │  · 从划线中提炼的    │
│  · 问题     │    核心知识点        │
│  · 提示     │  · 论据与例证       │
│  (1/3)     │  (2/3)             │
├────────────┴─────────────────────┤
│  总结栏 (Summary)                 │
│  · 用一两句话概括整页核心内容      │
└──────────────────────────────────┘
```

四个区域的作用：
- **标题栏**：书籍信息、章节、日期
- **线索栏**：提炼关键词、核心问题、记忆提示 —— 用于复习时自我测试
- **笔记栏**：基于划线整合的详细笔记
- **总结栏**：整页内容的一句话概括

## 工作流程

### 步骤 1：解析输入

首先识别用户提供的划线笔记来源格式：

**微信读书 Markdown 导出**（最常见）：
```markdown
# 书名
> 作者：作者名
◆ 章节名
>> 划线内容...
📝 想法：用户笔记...
```

**微信读书 JSON 导出**：
```json
{"bookName":"书名","author":"作者","notes":[{"chapter":"...","highlight":"...","note":"..."}]}
```

**Kindle My Clippings**：
```
==========
书名 (作者)
- 您在位置 #X-Y 的标注 | 添加于 YYYY-MM-DD
划线内容
==========
```

**纯文本**：用户直接粘贴的划线列表，自由格式。

如果格式无法识别，直接询问用户来源，或按纯文本处理。

### 步骤 2：内容预处理

在处理之前，对划线内容进行整理：

1. **去重**：移除完全重复的划线（同一句被多次划线）
2. **按章节分组**：如果原数据有章节信息，按章节组织；如果没有，按内容主题自动聚类
3. **区分划线与想法**：将原文划线与用户个人笔记/想法分开处理
   - 原文划线 → 进入「笔记栏」的素材
   - 用户想法 → 作为理解上下文，辅助 AI 提炼

### 步骤 3：AI 智能提炼

这是核心步骤。对每一章/每一组划线内容，进行三层提炼：

**第一层：关键词提取**
- 从划线中识别该章节的核心概念、术语、人名、方法论名称
- 每个章节提取 3-8 个关键词
- 关键词应该是「看到它就想起整段内容」的记忆触发器

**第二层：问题生成**
- 将核心知识点转化为「可测试的问题」
- 问题应该引导回忆，而非简单的是非题
- 好问题的特征：
  - ✅ 「为什么作者认为 X 是 Y 的关键原因？」
  - ❌ 「作者说了什么？」（太模糊）

**第三层：笔记整合**
- 将零散的划线整合为连贯的要点笔记
- 保留原文的关键论据、数据、例证
- 合并表达同一观点的多条划线
- 每条要点控制在 3-5 句以内，方便复习

### 步骤 4：撰写总结

为每个章节写一篇简短总结（3-5 句话）：
- 用自己（读者）的话重新表述，而非照搬原文
- 聚焦于「这一章的核心洞见是什么」
- 如果章节间有递进关系，在总结中标注

### 步骤 5：格式化输出

按以下 Markdown 模板输出。**严格遵循此模板结构**：

```markdown
# 📗 《{书名}》— 康奈尔笔记

> **作者**：{作者} | **日期**：{日期} | **总划线**：{N} 条

---

## {章节序号}. {章节名}

### 🔑 线索（Cue）— 复习自测

| # | 关键词 / 核心问题 |
|:---|:---|
| 1 | **{关键词}**：{一句话提示} |
| 2 | ❓ {可测试的问题}？ |
| 3 | **{关键词}**：{一句话提示} |

### 📝 笔记（Notes）— 详细内容

1. **{要点标题}**
   - {支撑细节，来自划线}
   - {例证或数据}

2. **{要点标题}**
   - {支撑细节}

### 📋 总结（Summary）

> {3-5 句话概括本章核心内容。用自己的话写，不是原文复述。}

---

## 📚 全书总览

| 章节 | 核心主题 | 关键概念 |
|:---|:---|:---|
| 第1章 | {一句话主题} | {关键词} |
| ... | ... | ... |

### 🎯 全书总结

> {整本书的核心洞见，2-3 句话。如果你读过相关书籍，可以建立联系。}
```

### 模板规则

- **书籍级**：一份康奈尔笔记对应一本书
- **章节级**：每个章节独立一页康奈尔笔记（重复「线索 → 笔记 → 总结」区块）
- **全书总览**：在所有章节之后，提供一个跨章节的汇总表
- 如果原文没有章节信息，按「主题聚类」代替章节，每个主题视为一个章节

## 处理原则

### 质量优先于数量
- 不是每条划线都要塞进笔记。AI 应判断划线的重要性，合并冗余，舍弃次要信息
- 宁可输出 5 条精炼要点，不要输出 20 条零散摘抄

### 理解优先于摘抄
- 笔记栏的内容应该是「理解后的重新表述」，而非原文照搬
- 如果划线和原文一模一样，说明提炼不到位
- 一个判断标准：读者看笔记时，能否不用翻原书就理解这个知识点？

### 问题驱动
- 线索栏的核心价值是「复习自测」—— 用户遮住右侧笔记栏，只看左侧问题，能否回忆起核心内容？
- 如果问题太模糊、太容易，或者不需要思考就能回答，就失去了价值

### 保留原始出处
- 在笔记要点末尾，用 `[@第3章-划线12]` 格式标注出处
- 这让用户可以回溯到原始划线，验证理解是否准确

### 处理用户想法/笔记
- 微信读书导出中的「📝 想法」是用户自己的思考
- 将其融入笔记或总结中，用 `💭 我的思考：...` 标注
- 用户的想法往往是最有价值的个人洞见，不要丢失

## 边缘情况处理

### 无章节信息
按内容主题自动聚类，每个聚类作为一个「章节」。在聚类标题前标注 `[自动分组]`。

### 划线过少（<5 条）
如果整本书划线很少，合并所有内容为一页康奈尔笔记，不强制分章节。
在总结中标注「本书划线较少，以下为整体笔记」。

### 划线过多（>100 条）
如果划线非常多，提供两种选项让用户选择：
1. **精选模式**：只取最有价值的 30-50 条进行整理，标注「已精选」
2. **完整模式**：逐章处理全部划线

默认使用精选模式，并在开头说明。

### 多本书混合
如果检测到输入包含多本书的内容（如微信读书导出了多本），
按书籍分开，每本书独立输出一份康奈尔笔记。

### 非中文内容
保持原语言输出。如果输入是英文，康奈尔笔记也用英文；中英混合则保持混合。

### 用户反馈循环
如果用户在看完输出后说「这部分不对」「这个词理解错了」，
先更新对应部分，再询问是否需要重新生成整份笔记。
